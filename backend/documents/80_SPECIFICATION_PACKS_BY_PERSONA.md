# Specification Packs by Persona

**Version:** 2026-06-18  
**Market:** South Africa / Southern Africa  
**Business:** Tiletoria / Tori RAG brain  
**Layer:** Phase 3 Control Layer  

## Purpose
Define what information different customer personas need when they ask for specifications, samples or project documentation.

## Retrieval triggers
- spec pack
- specification pack
- architect pack
- designer pack
- developer pack
- contractor pack
- QS pack
- insurance pack
- sample pack
- documents

## Why persona-specific spec packs matter
Different customers need different levels of information. A homeowner wants simple confidence. A QS wants quantities and pack sizes. An architect wants technical and aesthetic documentation. A contractor wants installation and availability clarity.

## Retail homeowner pack
Include or offer:

- product name and image if available
- size and finish
- indoor/outdoor/wet-area guidance if verified
- price/stock subject to branch confirmation
- m²/box and boxes required
- suggested adhesive/grout/trims
- cleaning and maintenance guidance
- showroom/branch handover

## DIY renovator pack
Include retail homeowner pack plus:

- measuring guide
- waste allowance explanation
- installation risk notes
- stop-work checks
- wet-area cautions
- what to ask the tiler/plumber

## Tiler/contractor pack
Include:

- product code/name
- size, thickness if available
- box m² and weight if available
- batch/shade/calibre warning
- adhesive recommendation category, subject to brand datasheet
- grout joint guidance
- levelling/spacer guidance
- substrate preparation notes
- stock/price confirmation route

## Plumber/sanitaryware pack
Include:

- product description
- technical spec sheet if available
- inlet/outlet compatibility notes
- waste/trap/mixer compatibility checks
- installation responsibility warning
- branch confirmation for parts and spares

## Designer pack
Include:

- product image/lifestyle if available
- colour/finish/style notes
- complementary options
- sample request route
- cleaning/maintenance expectations
- client approval checklist
- branch or Spec Lab routing

## Architect/specifier pack
Include:

- datasheet
- product code and description
- size, finish, rectified/non-rectified if available
- slip information if available
- application guidance
- installation system notes
- maintenance guidance
- warranty/limitation document if approved
- sample/board request route

## QS/procurement pack
Include:

- product code
- UOM
- m²/box
- minimum order/special order note if relevant
- current price subject to quote
- stock/lead time subject to confirmation
- alternative options
- waste factor guidance
- delivery/collection notes

## Developer pack
Include architect and QS requirements plus:

- range continuity risk
- batch planning
- supply phasing
- substitution approval process
- bulk delivery planning
- site storage notes

## Insurance replacement pack
Include:

- existing product details/photos
- approximate m² and area damaged
- like-for-like or practical substitute requirement
- insured/customer/contractor/assessor details
- branch/region
- product availability subject to confirmation
- extras required for reinstatement

## Tori response template
> I can prepare the right type of information pack. Please confirm whether this is for a homeowner, tiler, contractor, designer, architect, QS, developer or insurance claim, and share the product name/code if available.

---

## Universal Tori rule for this file
Tori must be helpful, practical and commercially safe. Tori may guide, calculate, explain, collect details and prepare a handover, but must not invent live stock, live prices, delivery availability, technical certifications, warranties, legal outcomes, account status, branch decisions or product suitability where site conditions are unknown. When the answer depends on current branch systems, ERP data, site inspection, supplier confirmation or management approval, Tori must clearly say so and route the customer to the correct Tiletoria team.
